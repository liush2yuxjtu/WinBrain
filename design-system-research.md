## ChatGPT 设计系统

### 概览

ChatGPT 采用简洁、现代的设计语言,强调内容可读性和交互流畅性。

### 配色方案

| 类型 | 用途 | 色值示例 | 说明 |
|------|------|----------|------|
| **主色调** | 品牌色、主要按钮 | `#10A37F` (深绿) | OpenAI 品牌绿色 |
| **辅助色** | 次要交互元素 | `#19C37D` (亮绿) | hover 状态 |
| **中性色-深** | 文本、边框 | `#202123` | 主要文本 |
| **中性色-中** | 次要文本 | `#6E6E80` | 辅助文本 |
| **中性色-浅** | 背景、分割线 | `#F7F7F8` | 背景色 |
| **白色** | 内容区域 | `#FFFFFF` | 对话气泡 |
| **错误色** | 错误状态 | `#EF4444` | 错误提示 |

### 字体系统

| 层级 | 字体族 | 大小 | 行高 | 字重 |
|------|--------|------|------|------|
| **H1** | Söhne/system-ui | 32px | 1.2 | 600 |
| **H2** | Söhne/system-ui | 24px | 1.3 | 600 |
| **正文** | Söhne/system-ui | 16px | 1.6 | 400 |
| **代码** | Source Code Pro | 14px | 1.6 | 400 |

### 布局模式

**三栏布局**: 侧边栏(240px) + 主区域(flex) + 右栏(280px)
**间距系统**: 4px 基础单位,常用 8/12/16/24/32/48px
**圆角**: 小4px, 中8px, 大12px
**阴影**: 分层使用,轻微到深度

## Claude 设计系统

### 概览

Claude 的设计系统注重专业性和清晰度,提供企业级的视觉体验。采用温暖的中性色调和清晰的层次结构。

### 配色方案

| 类型 | 用途 | 色值示例 | 说明 |
|------|------|----------|------|
| **主色调** | 品牌色、强调 | `#CC785C` (陶土橙) | Anthropic 品牌色 |
| **辅助色** | 次要交互 | `#E09D88` (浅陶土) | hover 状态 |
| **中性色-深** | 文本 | `#1A1A1A` | 主要文本 |
| **中性色-中** | 次要文本 | `#6B6B6B` | 辅助文本 |
| **中性色-浅** | 背景 | `hsl(60,11%,95%)` | 页面背景 |
| **白色** | 内容区域 | `#FFFFFF` | 对话区域 |
| **蓝色** | 链接、信息 | `#0066CC` | 链接色 |

**主题色**: `hsl(60,11%,95%)` - 温暖的米色背景

### 字体系统

| 层级 | 字体族 | 大小 | 行高 | 字重 |
|------|--------|------|------|------|
| **H1** | system-ui | 36px | 1.2 | 600 |
| **H2** | system-ui | 28px | 1.3 | 600 |
| **正文** | system-ui | 16px | 1.6 | 400 |
| **代码** | monospace | 14px | 1.6 | 400 |

**字体栈**:
```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
```

### 布局模式

**双栏布局**: 侧边栏(280px) + 主内容区(flex)
**主内容最大宽度**: 720px (居中对齐)
**间距系统**: 8px 基础单位,常用 16/24/32/48px
**圆角**: 统一使用 8px
**阴影**: 细微柔和的阴影,强调层次

## 设计系统对比

### 配色对比

| 维度 | ChatGPT | Claude | 分析 |
|------|---------|--------|------|
| **品牌色** | 绿色系 (#10A37F) | 橙色系 (#CC785C) | ChatGPT 更科技感,Claude 更温暖 |
| **背景色** | 冷灰色 (#F7F7F8) | 暖米色 (hsl(60,11%,95%)) | Claude 更温和,减少眼睛疲劳 |
| **对比度** | 高对比 | 中高对比 | ChatGPT 更鲜明,Claude 更柔和 |
| **色彩数量** | 较多 | 较少 | ChatGPT 更丰富,Claude 更克制 |

### 字体对比

| 维度 | ChatGPT | Claude |
|------|---------|--------|
| **品牌字体** | Söhne (定制) | 系统字体 |
| **正文大小** | 16px | 16px |
| **行高** | 1.6 | 1.6 |
| **特点** | 品牌化强 | 通用性强 |

### 布局对比

| 维度 | ChatGPT | Claude |
|------|---------|--------|
| **主结构** | 三栏可选 | 双栏固定 |
| **侧边栏** | 240px,可折叠 | 280px |
| **内容宽度** | 768px | 720px |
| **复杂度** | 较高(多功能) | 简洁(聚焦对话) |

### 共同特点

1. **极简主义**: 两者都采用简洁设计,减少视觉干扰
2. **高可读性**: 大字号(16px)、适当行高(1.6)
3. **响应式**: 都支持移动端适配
4. **无障碍**: 注重对比度和键盘导航
5. **空白运用**: 大量留白提升可读性

## 最佳实践总结

### 1. 配色原则

**以中性色为主,品牌色点缀**
- 背景和主体使用中性灰/米色(占比 80%)
- 品牌色仅用于关键交互点(占比 10-15%)
- 保持 4.5:1 以上的对比度(WCAG AA 标准)

**推荐方案**:
```css
--bg-primary: #FAFAFA;      /* 主背景 */
--bg-secondary: #FFFFFF;    /* 内容背景 */
--text-primary: #1A1A1A;    /* 主文本 */
--text-secondary: #6B6B6B;  /* 次要文本 */
--brand-primary: #自定义;    /* 品牌主色 */
```

### 2. 字体系统原则

**可读性优先**
- 正文字号: 16px (最小 14px)
- 行高: 1.5-1.6 倍
- 字重: 正文 400,标题 600
- 字符间距: 正常或略宽松 (0-0.02em)

**字体栈策略**:
- 优先使用系统字体(加载快,体验好)
- 品牌字体作为可选增强
- 代码块使用等宽字体

### 3. 布局原则

**内容宽度控制**
- 聊天内容: 650-768px (最佳阅读宽度)
- 每行字符数: 60-80 (中文 40-50)
- 侧边栏: 240-280px

**间距系统**
- 采用 4px 或 8px 基础单位
- 使用倍数关系: 8, 16, 24, 32, 48, 64px
- 组件内部间距小,组件之间间距大

**层次表达**
- Z 轴分层: 背景(0) → 内容(1) → 浮层(10) → 模态(100)
- 阴影深浅表达高度
- 避免过度使用阴影

### 4. 交互原则

**视觉反馈**
- Hover: 颜色变化或轻微放大
- Active: 颜色加深或轻微缩小
- Focus: 明显的焦点环(可访问性)
- Loading: 骨架屏或加载动画

**动画时长**
- 快速交互: 150-200ms
- 过渡动画: 250-300ms
- 复杂动画: 400-600ms
- 缓动函数: ease-out (大多数场景)

### 5. 空白运用原则

**慷慨的留白**
- 对话气泡间距: 12-16px
- 段落间距: 16-24px
- 区块间距: 32-48px
- 页面边距: 24-32px (移动端 16px)

**视觉呼吸感**
- 避免内容过于拥挤
- 重要元素周围留出足够空间
- 用空白引导视觉流

### 6. 响应式原则

**断点建议**
- 移动端: < 640px
- 平板: 640px - 1024px
- 桌面: > 1024px
- 大屏: > 1440px

**适配策略**
- 移动端: 单栏布局,侧边栏折叠
- 平板: 可选侧边栏
- 桌面: 完整三栏布局

### 7. 简化设计的核心要点

**删除非必要元素**
- 只保留核心功能可见
- 次要功能隐藏在菜单中
- 避免过度装饰

**建立清晰的视觉层次**
- 大小对比明显(标题 vs 正文)
- 颜色对比区分层级
- 间距表达归属关系

**一致性**
- 相同功能使用相同样式
- 相同层级使用相同大小
- 建立并遵守设计规范

### 8. 针对 AI 聊天界面的特殊考虑

**长内容优化**
- 支持代码高亮
- 表格和列表清晰展示
- Markdown 渲染美观

**交互便利性**
- 复制按钮显而易见
- 消息编辑和重新生成
- 历史记录易于访问

**性能考虑**
- 长对话虚拟滚动
- 图片懒加载
- 动画性能优化
